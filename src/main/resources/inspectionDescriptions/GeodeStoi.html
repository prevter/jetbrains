Reports usage of std::stoi and similar string-to-number conversion functions in C++ code.

These functions (stoi, stol, stoll, stoul, stoull, stof, stod, stold) are rejected by the Geode mod index and should be replaced with geode::utils::numFromString for better error handling.

Example: 


// Bad - will be rejected
int value = std::stoi(input);

// Good - use Geode's safe conversion
auto result = geode::utils::numFromString<int>(input);
if (result.isOk()) {
    int value = result.unwrap();
}

The Geode SDK provides numFromString as a safer alternative that returns a Result type for proper error handling.
